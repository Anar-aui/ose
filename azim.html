Git workflow — это способ работы с Git, который помогает команде организовать изменения в коде. Он описывает, как создавать и объединять ветки (версии кода),я понял, чтобы разработка была упорядоченной и удобной

git merge: Сохраняет историю всех коммитов обеих веток и создаёт новый "слияющий" коммит. История остаётся "ветвистой", показывая, где и когда ветки объединялись.

git rebase: Перемещает (переписывает) коммиты одной ветки на вершину другой, делая историю линейной, без ветвлений. Это как будто все изменения происходили последовательно.

Git fetch — это команда, которая загружает обновления из удалённого репозитория (например, на GitHub) на ваш компьютер, но не вносит изменения в ваш текущий код. Это похоже на то, как если бы вы проверяли, появились ли новые версии файлов, не вмешиваясь в свою работу.
